# 修复发票字段

## 运行环境

- 所有命令都从项目根目录运行。
- 禁止使用 Python 或自编脚本提取、推断、修复字段。只允许 Read 直接查看 PDF，以及使用下文明确给出的 `pdftotext` 命令辅助读取。

读取 `报销工作文件/invoice_errors_raw.json`，只处理其 `errors[]` 中明确列出的字段。使用 `pdftotext -layout` 读取 `invoices/` 中对应发票；打车发票同时核对行程单。无法从原始文件可靠确定时停止，不猜测。

不要添加、删除或重排 `项目列表` 条目，不修改错误列表之外的字段。

将修正写入 `报销工作文件/invoice_fixes.json`。输出结构必须是“发票文件名 → 需要修正的字段 → 正确值”。外层键必须原样使用错误列表中的 `文件名`，内层键必须原样使用错误列表中的 `字段`，内层值必须替换为从对应 PDF 或行程单可靠提取的正确值。最终文件只能包含错误列表中需要修正的字段，不得保留示例占位符或照搬示例中的其他字段。

```json
{
  "example.pdf": {
    "购买方税号": "12100000470095016Q",
    "价税合计金额": 39.04,
    "项目列表.0.项目名称": "交通运输服务",
    "项目列表.0.单价": 20.49,
    "开票时间": [
      {
        "年": 2026,
        "月": 7,
        "日": 1
      }
    ],
    "发票号码状态": "正常"
  }
}
```

严格遵守以下 JSON 类型：

- `价税合计金额` 以及路径以 `.单价` 结尾的字段必须是非负 JSON number，禁止加引号。
- `开票时间` 必须是仅包含日期对象的 JSON array；每个日期对象必须包含 `年`、`月`、`日`，且三者均为 JSON integer。
- 税号、名称、发票号码、项目名称和状态字段必须是 JSON string。
- `发票号码状态` 只能是 `正常` 或 `需人工校验`。
- 写入后重新读取 `invoice_fixes.json`，逐字段确认键名来自错误列表且值符合上述类型；任何字段无法满足时立即停止并报告，不要输出不完整或类型不正确的修复文件。

完成后仅向主流程报告修复的发票数和字段数。
